Objetivos

Conoce nuestros objetivos generales y específicos que guían este proyecto

🎯

OBJETIVO GENERAL DEL PROYECTO

A la hora de trabajar con una tienda online de zapatillas y una cantidad amplia de productos y datos de clientes, es de vital importancia una buena organización para evitar errores y confusiones. Por esa misma razón se plantea una aplicación web para gestionar el trabajo y facilitar las tareas lo máximo posible.

Por una parte tenemos la vista de las zapatillas, donde se mostrarán sus datos principales: nombre del modelo, marca y precio. El administrador podrá gestionar las fotos del producto que se mostrarán en la página. También aparecerá una descripción detallada que podrá añadir el administrador, como por ejemplo el material, características especiales o algún dato de interés relevante. El administrador podrá modificar, añadir o borrar cualquier característica de las zapatillas, con especial atención al control del stock disponible.

Toda la gestión administrativa que lleva el proceso de compra y venta también estaría implementada en la aplicación. Como por ejemplo, la gestión de pedidos, que es el proceso previo de confirmación y preparación del producto para su envío rápido también estaría dentro.

OBJETIVOS ESPECÍFICOS

Nuestro proyecto será una aplicación web la cual se guardará los datos en una base de datos relacional MySQL. La aplicación tendrá un apartado gráfico muy intuitivo y fácil de usar para más comodidad del cliente.

En nuestra aplicación web habrá dos niveles de usuario, es decir, cada usuario tiene su propio acceso a la aplicación, pero todos tendrán el mismo nivel de usuario con las mismas funcionalidades excepto uno. Este usuario tendrá el usuario administrador, con el que podrá administrar a los distintos usuarios de la plataforma.

1

Vista de Administrador - Gestión de Stock

Si se inicia la sesión con el usuario de administrador, este podrá gestionar el stock de las zapatillas, presentadas en una tabla con su nombre del modelo, marca y precio. Podrá modificar las cantidades disponibles, añadir nuevos productos o eliminar los que ya no estén disponibles. También podrá gestionar los usuarios registrados en la plataforma.

2

Vista de Usuario - Catálogo y Compras

Los usuarios al entrar verán las zapatillas, y si hacen clic en ellas podrán seleccionar talla y más adelante comprar una zapatilla. Podrán navegar por todo el catálogo de productos de forma intuitiva y atractiva.

3

Gestión de Pedidos y Órdenes

Se guarda una orden por cada cliente que haya comprado una zapatilla. El sistema registrará todas las transacciones y mantendrá un historial completo de las compras realizadas por cada usuario.

4

Sistema de Pagos Seguro

Cada cliente tiene asociada una tarjeta de crédito la cual se valida para hacer la compra. Se implementará un sistema de pago seguro que proteja los datos financieros de los usuarios durante las transacciones.

5

Base de Datos MySQL Centralizada

Todo el sistema estará respaldado por una base de datos MySQL que almacenará información de usuarios, productos, pedidos y datos de pago, garantizando la integridad y consistencia de los datos.

6

Consultas y Filtros Avanzados

Los filtros y/o consultas se realizan mediante queries para obtener un listado óptimo y fácil de leer que pueda consultar el cliente, permitiendo búsquedas eficientes de productos por marca, talla, precio, etc.
